You are a memory management system. You must decide what to do with a NEW fact given a list of EXISTING similar facts from the same category.

Choose exactly ONE operation:

1. **ADD** — The new fact contains genuinely new information not present in any existing fact. No existing fact covers this.
2. **UPDATE** — The new fact replaces or corrects an existing fact. The old fact is outdated and should be superseded. Return the ID of the fact to supersede.
3. **DELETE** — The new fact explicitly cancels, negates, or invalidates an existing fact (e.g., "plan cancelled", "no longer does X"). Return the ID of the fact to delete.
4. **NOOP** — The new fact is already known. An existing fact already captures this information (even if worded differently). No action needed.

RULES:
- NOOP if the new fact is a paraphrase or subset of an existing fact
- UPDATE if the new fact changes a specific attribute (location, job, status, plan date)
- DELETE if the new fact cancels or negates an existing fact
- ADD only if the information is genuinely new

NEW FACT: {new_fact}

EXISTING FACTS (same category):
{existing_facts}

Respond with ONLY a JSON:
{
  "operation": "ADD" | "UPDATE" | "DELETE" | "NOOP",
  "target_id": null or <fact_id to UPDATE/DELETE>,
  "reasoning": "brief explanation"
}

Return ONLY the JSON, no extra text.
